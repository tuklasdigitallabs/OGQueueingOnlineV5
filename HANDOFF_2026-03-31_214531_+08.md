# QSys Online Handoff

Updated on: 2026-04-13

Where we stopped:
- Staff business date rollover was fixed. `ensureBusinessDate()` now advances stale persisted dates to the current Manila date on request, not only through the 30-second watcher.
- Guest ticket page now shows the guest's current waiting position within their specific pax group. It is computed live from `/api/guest/ticket` and updates when queue state changes/refetches.
- Display media upload now uses FFmpeg conversion. Admin uploads common video files, then the server saves a web-safe MP4 (`H.264`, `AAC`, `yuv420p`, even dimensions, `faststart`).
- Docker image now installs `ffmpeg` so VPS media uploads can transcode inside `og-qsys-app`.
- Cloud media playback was fixed for the local Electron display by allowing cross-origin media responses (`Cross-Origin-Resource-Policy: cross-origin`, `Access-Control-Allow-Origin: *`).
- Display debug logs were quieted and the portrait close button now tries the local Electron bridge before the staff-authenticated close endpoint.
- Branch licensing/activation was added. Existing active branches are initialized as activated for migration safety, but newly created or CSV-imported branches start `UNACTIVATED`.
- Provider Setup now lists branch license status and includes branch Activate/Renew token actions.
- Unactivated branches are blocked from staff view, guest registration, public branch listing, staff queue APIs, and display/media access.
- Admin > Branches now shows license status separately from branch record status.
- Admin > Media now includes "Clean Old Files" controls for missing/pre-conversion managed media records.
- Local file media selection in the Electron launcher now converts the chosen file to a web-safe MP4 copy before playback.
- Electron display `webSecurity: false` was removed, and display pages now include a CSP allowing the local app, cloud media, and websocket/fetch connections.
- FFmpeg CLI bundling was prepared for the Windows installer:
  - `package.json` bundles `vendor/ffmpeg/win/ffmpeg.exe` into `resources/ffmpeg/ffmpeg.exe`.
  - `electron/main.js` resolves bundled FFmpeg first, then dev/vendor paths, then system `ffmpeg`.
  - `scripts/prepare_ffmpeg_windows.ps1` can download FFmpeg for the build machine.
  - User manually downloaded `ffmpeg.exe` and placed it in `vendor/ffmpeg/win/ffmpeg.exe`.
- GitHub Actions CI/CD was added:
  - `.github/workflows/ci.yml` validates pushes/PRs and builds `Dockerfile.qsys`.
  - `.github/workflows/deploy-vps.yml` deploys to the VPS over SSH after successful `main` CI or via manual dispatch.
  - Duplicate legacy workflow `.github/workflows/deploy-qsys.yml` was removed to avoid secret-name and behavior drift.
